# 🚀 PHP Smart Files v1.1.0 Release Notes  
*Released on 2025-07-07*

## ✨ New Features

- 🎉 Added support for `use function` and `use const` imports during namespace refactoring  
  - All PHP import types (classes, functions, constants) are properly recognized and updated when files are moved or renamed.  
  - Configurable via settings `phpSmartFiles.refactorNamespacesIncludeFunctions` and `phpSmartFiles.refactorNamespacesIncludeConstants`.

- 🔀 Automatic sorting of `use` statements during refactoring  
  - Groups imports by type (`use`, `use function`, `use const`) and sorts them alphabetically within groups.  
  - Can be toggled on/off with `phpSmartFiles.sortUseStatementsOnRefactor` (default: enabled).  
  - Keeps imports clean and consistent after refactoring.

- 🛡️ Introduced `GlobalReservedService`  
  - Prevents naming conflicts by detecting globally reserved PHP names from runtime, dependencies, and frameworks.  
  - Avoids invalid `use` statements and improves validation during refactoring and code generation.

- ➕ Added ability to create PHP functions via the new file creation dialog, alongside classes, interfaces, traits, and enums.

## 🔧 Improvements

- Renaming files to globally reserved names now prompts a clear warning and requires explicit confirmation to avoid accidental conflicts.  
- Confirmation dialogs after file or directory renames/moves are now informational messages instead of warnings, improving user experience.  
- Updated extension logo and marketplace banner color for a fresh, consistent visual identity.
